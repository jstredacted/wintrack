## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between text and icon |
| sm | 8px | Compact element spacing, tab trigger padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, overlay content padding |
| xl | 32px | Layout gaps, page-level section spacing |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets: all interactive elements on mobile minimum 44x44px (MOB-03). DayStrip cells, tab bar items, checkboxes, and NavLinks must meet this. Achieved via `min-h-[44px] min-w-[44px]` or `p-[10px]` padding additions where needed.
- Bottom tab bar height: `3.5rem` (56px at 18px base) + `env(safe-area-inset-bottom)` (~34px on iPhone). Total visual height on iPhone ~90px — AppShell main area must clear this with `calc(3.5rem + env(safe-area-inset-bottom))` bottom padding on mobile.
- Slash command menu: 4px vertical padding per item, 8px horizontal. Menu max-width 180px, minimum width 140px.

## Typography

Base font size is 18px on `html` element. All sizes below are at 18px scale (1rem = 18px).

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Label/caption | 10px (fluid-sm ≈ 0.625–0.8125rem) | `text-[10px]` or `text-xs` | 400 regular | 1.4 | Bottom tab bar labels, badge counts, metadata |
| Body | 16px (0.889rem) | `text-sm` | 400 regular | 1.5 | Paragraph text, form inputs, card content |
| UI / default | 18px (1rem) | `text-base` | 400 regular | 1.5 | Default body copy, settings rows, list items |
| Heading | ~25px (1.4rem) | `text-2xl` | 600 semibold | 1.2 | Section headings, Tiptap H2 output |
| Display | ~36px (2rem) | `text-4xl` | 700 bold | 1.1 | Page titles, hero numerals |

Font weights: 400 (regular) and 600 (semibold) only. 700 (bold) permitted only for Tiptap H2 heading output and display numerals — not for UI chrome.

Font face: Geist Mono Variable for all roles. The project has both `--font-sans` and `--font-mono` set to Geist Mono — everything is monospace.

### Tiptap Editor Typography

Tiptap's ProseMirror output is scoped under `.tiptap-editor`. These sizes apply inside the editor and in `JournalEntryCard` HTML renders:

| Element | Size | Weight | Line Height |
|---------|------|--------|-------------|
| Body paragraph | 18px (1rem base) | 400 | 1.7 (more generous for reading) |
| H2 | 1.4rem = ~25px | 700 | 1.2 |
| H3 | 1.15rem = ~21px | 600 | 1.3 |
| Bullet / list items | 1rem | 400 | 1.6 |
| Placeholder | 1rem | 400 | — (muted-foreground at opacity 0.3) |

## Color

Project uses the Nova preset with neutral (achromatic) color tokens. All tokens are CSS variables defined in `src/index.css`. There is no chromatic accent — the "Nothing Phone" aesthetic is intentionally monochrome.

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `--background` oklch(1 0 0) white | oklch(0.145 0 0) near-black | Page background, main surface |
| Secondary (30%) | `--card` oklch(1 0 0) | oklch(0.205 0 0) dark gray | Cards, modal backgrounds, sidebar |
| Accent (10%) | `--accent` oklch(0.97 0 0) | oklch(0.371 0 0) | See reserved list below |
| Muted | `--muted` oklch(0.97 0 0) | oklch(0.269 0 0) | Inactive states, dividers, empty cells |
| Destructive | `--destructive` oklch(0.58 0.22 27) red | oklch(0.704 0.191 22.216) | Overdue bill urgency borders only |

Accent (`--accent`) reserved for:
1. Active tab indicator — `border-b-2 border-foreground` on `TabsTrigger[data-state=active]` (uses foreground, not accent token)
2. DayStrip selected day — `border-b-2 border-foreground` on active cell
3. Bottom tab bar active item — foreground color on active NavLink icon + label
4. Slash command menu hover/focus row — `bg-accent text-accent-foreground`
5. Settings tab active state — underline in foreground color

Note: This project uses foreground-as-accent for selection states, not a chromatic color. The `--accent` token (`oklch(0.97 0 0)` light / `oklch(0.371 0 0)` dark) is the hover/focus background. Selected state is always `border-foreground` or `text-foreground`.

## Interaction Contracts

### Rich Text Formatting Discovery (three paths)

| Path | How |
|------|-----|
| Keyboard shortcuts | Cmd+B (bold), Cmd+I (italic), Cmd+Z (undo) — built into StarterKit, no custom code |
| Markdown input rules | `**text**` → bold, `_text_` → italic, `## ` → H2, `### ` → H3, `- ` → bullet, `1. ` → numbered — built into StarterKit |
| Slash commands | Type `/` → floating menu appears → arrow keys or click to select → Enter to confirm |

### DayStrip Selection and Centering

| Trigger | Behavior |
|---------|----------|
| Mount | Scroll DayStrip to center the selected day (not rightmost) |
| `selectedDate` prop change | Scroll to re-center new selected date |
| Cell tap/click | Select that date (existing behavior) |
| Centering formula | `scrollLeft = cellOffsetLeft + cellWidth/2 - containerWidth/2` |
| Fallback | If selected cell not found, scroll to rightmost |

### Rollover Prompt

Only shows wins from the previous logical day that are NOT yet completed (`completed: false` filter at DB query level). Completed wins never appear in rollover prompt.

## Animation Contracts

All overlay animations use plain `@keyframes` in `src/index.css` — NOT motion library (existing project pattern).

| Animation | Keyframe | Timing | Usage |
|-----------|----------|--------|-------|
| Journal overlay entry | `journal-slide-in` (translateY 100vh→0) | 0.35s cubic-bezier(0.32, 0.72, 0, 1) | JournalEditorOverlay open |
| Journal overlay exit | `journal-slide-out` (translateY 0→100vh) | 0.22s cubic-bezier(0.4, 0, 1, 1) | JournalEditorOverlay close |
| Slash menu | Instant appear/disappear (no animation) | — | Keeps editor feel snappy |
| Bottom tab bar | No animation on appear — it is always present on mobile | — | — |
| Settings tabs | Instant content switch (no cross-fade) | — | Consistent with existing tab pattern |

Critical constraint: `<EditorContent editor={editor} />` must sit inside a container that is NOT animated with CSS `translate` during entry. The outer overlay wrapper can use `journal-slide-in` keyframes (uses `transform: translateY`). The inner content area must be a static positioned div. If an inner div must animate, use `opacity` only.

## CSS Additions Required

Add to `src/index.css`:

```css
/* Tiptap editor scoped styles */
.tiptap-editor .ProseMirror {
  outline: none;
  min-height: 200px;
}

.tiptap-editor .ProseMirror p.is-editor-empty:first-child::before {
  content: attr(data-placeholder);
  color: var(--muted-foreground);
  opacity: 0.3;
  float: left;
  height: 0;
  pointer-events: none;
}

.tiptap-editor h2 { font-size: 1.4rem; font-weight: 700; margin: 1rem 0 0.5rem; line-height: 1.2; }
.tiptap-editor h3 { font-size: 1.15rem; font-weight: 600; margin: 0.75rem 0 0.25rem; line-height: 1.3; }
.tiptap-editor p { line-height: 1.7; margin-bottom: 0.5rem; }
.tiptap-editor ul, .tiptap-editor ol { padding-left: 1.5rem; }
.tiptap-editor li { margin: 0.2rem 0; line-height: 1.6; }

/* Mobile bottom nav clearance */
.mobile-bottom-clearance {
  padding-bottom: calc(3.5rem + env(safe-area-inset-bottom));
}
@media (min-width: 640px) {
  .mobile-bottom-clearance { padding-bottom: 0; }
}
```

Do NOT use `@tailwindcss/typography` prose classes — they conflict with Tiptap inline styles.
