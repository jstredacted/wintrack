3. Run `bun run typecheck` (which runs `tsc --noEmit`) and fix all errors. Common strict-mode errors to expect:
   - **Implicit any parameters:** Add explicit types to function parameters (e.g., `(e)` -> `(e: React.FormEvent<HTMLFormElement>)`).
   - **Possibly null/undefined:** Add null checks or use optional chaining. For example, `ref.current.focus()` -> `ref.current?.focus()`.

   - **No implicit this:** Should not be an issue in React functional components.

4. DO NOT use `any` to fix errors. Use:
   - Proper types from `Database['public']['Tables'][...]`
   - `unknown` with type narrowing
   - React built-in types (`React.ReactNode`, `React.MouseEvent`, etc.)
   - Generic constraints where needed
