  <files>src/hooks/useStreak.js</files>
  <behavior>
    - journalStreak: 0 when no journal entries exist
    - journalStreak: 1 when today has at least one journal entry
    - journalStreak: N for N consecutive days with ≥1 entry (deduplication via Set)
    - Multiple entries same day counts as streak: 1 (not 2)
    - loading remains false after both streak and journalStreak are resolved
    - Existing streak tests remain GREEN
  </behavior>
  <action>
Modify src/hooks/useStreak.js to:
1. Add `journalStreak` state: `const [journalStreak, setJournalStreak] = useState(0);`
2. Inside `fetchStreak`, after the existing wins streak computation, add a second query:
   ```js
   const { data: journalData } = await supabase
     .from('journal_entries')
     .select('created_at')
     .eq('user_id', USER_ID);

   const journalDates = new Set(
     (journalData ?? []).map(row => getLocalDateString(new Date(row.created_at)))
   );

   let jCount = 0;
   let jCursor = new Date();
   while (journalDates.has(getLocalDateString(jCursor))) {
     jCount++;
     jCursor = new Date(jCursor.getTime() - 86400000);
   }
   setJournalStreak(jCount);
   ```
3. Move `setLoading(false)` to after both streak computations complete.
4. Update the return value to: `return { streak, journalStreak, loading };`

The `cancelled` guard should wrap the entire result: `if (cancelled) return;` stays where it is — it already covers both computations since they run sequentially in the same async function.

The existing `buildStreakMock` in the test file uses `supabase.from.mockReturnValue(...)`. The new tests use `supabase.from.mockReturnValueOnce(...).mockReturnValueOnce(...)` so the two sequential `.from()` calls get different mock responses — wins streak first, journal streak second. This matches the sequential query order in the hook.
  </action>
