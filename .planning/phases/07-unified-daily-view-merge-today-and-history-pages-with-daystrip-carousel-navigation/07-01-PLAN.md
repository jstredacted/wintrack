   - `const isToday = selectedDate === today`

3. Add history hooks alongside existing useWins:
   - `const { completionMap, fetchWinsForDate } = useHistory()`
   - State for past-date wins: `const [historyWins, setHistoryWins] = useState([])` and `const [detailLoading, setDetailLoading] = useState(false)`

4. Add useEffect to fetch past-date wins (only when `!isToday`):
   ```jsx
   useEffect(() => {
     if (isToday) return;
     let cancelled = false;
     setDetailLoading(true);
     fetchWinsForDate(selectedDate).then(data => {
       if (!cancelled) { setHistoryWins(data); setDetailLoading(false); }
     });
     return () => { cancelled = true; };
   }, [selectedDate, isToday, fetchWinsForDate]);
   ```

5. Merge today's completion into completionMap for live DayStrip updates:
   ```jsx
   const todayHasCompleted = wins.some(w => w.completed);
   const mergedCompletionMap = { ...completionMap, [today]: todayHasCompleted || undefined };
   ```
   Note: `|| undefined` ensures if no wins are completed, the key is removed (not set to false) so DayStrip shows no checkmark.

6. Layout structure:
   - Keep the greeting header as-is but only show when `isToday`. When `!isToday`, show the selected date formatted nicely as the heading instead (e.g., format with `Intl.DateTimeFormat('en-US', { weekday: 'long', month: 'long', day: 'numeric' })`).
   - DayStrip rendered OUTSIDE any conditional block (always mounted, never unmounts on re-render -- per research pitfall 2). Place it after the header, in its own edge-to-edge bleed container: `<div className="mb-10 -mx-16 px-16 border-y border-border/30 py-4">` matching HistoryPage's current styling.
   - Content area below DayStrip: conditionally render based on `isToday`:
     - `isToday`: existing win list content (AnimatePresence loading/content, WinList, CategorySummary, Set intentions button, RollForwardPrompt, WinInputOverlay)
     - `!isToday`: `<DayDetail date={selectedDate} wins={historyWins} loading={detailLoading} />` wrapped in AnimatePresence with key={selectedDate} for crossfade on date change. Use AnimatePresence WITHOUT mode="wait" (per Phase 03-03 decision -- jsdom blocks on mode="wait").

7. Hide today-only elements when `!isToday`: RollForwardPrompt, Set intentions button, WinInputOverlay should only render when `isToday`.
