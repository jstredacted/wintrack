Set up the foundational push notification infrastructure: service worker with push event handling, vite-plugin-pwa configuration, push subscription client library with Supabase persistence, and the database migration for storing push subscriptions.

Purpose: This is the core plumbing that all push notification features depend on. The service worker must be registered and listening for push events before any notifications can be delivered. The subscription must be stored server-side so the Edge Function (Plan 03) can send to it.

Output: Working service worker registration, push subscription management library, DB table for subscriptions, and replaced notification stubs.

<task type="auto">
  <name>Task 2: Create push-subscription library and replace notification stubs</name>
  <files>src/lib/push-subscription.js, src/lib/notifications.js</files>
  <action>
1. Create `src/lib/push-subscription.js`:
   - Import `supabase` from '@/lib/supabase' and `USER_ID` from '@/lib/env'
   - Read VAPID public key: `const VAPID_PUBLIC_KEY = import.meta.env.VITE_VAPID_PUBLIC_KEY`
   - Implement `urlBase64ToUint8Array(base64String)` helper: pad to multiple of 4 with '=', replace `-` with `+` and `_` with `/`, atob decode, convert to Uint8Array
   - Export `async function subscribeToPush()`:
     - `const registration = await navigator.serviceWorker.ready`
     - `const subscription = await registration.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(VAPID_PUBLIC_KEY) })`
     - `const json = subscription.toJSON()`
     - Upsert to Supabase: `supabase.from('push_subscriptions').upsert({ user_id: USER_ID, endpoint: json.endpoint, p256dh: json.keys.p256dh, auth: json.keys.auth, expiration_time: json.expirationTime }, { onConflict: 'user_id' })`
     - Return subscription
   - Export `async function unsubscribeFromPush()`:
     - Get existing subscription via `registration.pushManager.getSubscription()`
     - If exists, call `subscription.unsubscribe()` and `supabase.from('push_subscriptions').delete().eq('user_id', USER_ID)`
   - Export `async function getExistingSubscription()`:
     - Returns `registration.pushManager.getSubscription()` (or null if no SW)

2. Replace `src/lib/notifications.js` entirely:
   - Import `subscribeToPush` from './push-subscription'
   - `export async function scheduleMorningReminder()`: call `subscribeToPush()` — actual scheduling is done server-side by pg_cron, client just needs to be subscribed
   - `export async function scheduleEveningReminder()`: same — these are now no-ops that ensure subscription exists (idempotent)
   - Add comment explaining: "Notification timing is handled server-side by pg_cron + Edge Function. These functions ensure the browser is subscribed to push. Actual delivery happens via Supabase."
