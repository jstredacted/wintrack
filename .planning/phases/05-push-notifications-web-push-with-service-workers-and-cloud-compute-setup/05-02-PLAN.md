<objective>
Add the user-facing notification permission UI: a React hook for push subscription state, a NotificationPermission toggle component, and integration into the Settings page.

Purpose: Users need a clear, accessible way to opt in/out of push notifications. The permission request must happen on a user gesture (click), not on page load, to avoid browser blocking. The Settings page already has the morning/evening hour controls -- adding the notification toggle here creates a single place for all notification preferences.

Output: NotificationPermission component rendered on Settings page, usePushSubscription hook for reactive state.
</objective>
