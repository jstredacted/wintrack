**008_finance_tables.sql** — Create three tables:

1. `months` table:
   - `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id uuid NOT NULL`
   - `year int NOT NULL CHECK (year >= 2000 AND year <= 3000)`
   - `month int NOT NULL CHECK (month BETWEEN 1 AND 12)`
   - `starting_balance numeric(14,2) NOT NULL DEFAULT 0`
   - `current_balance numeric(14,2) NOT NULL DEFAULT 0`
   - `budget_limit numeric(14,2) NOT NULL DEFAULT 0 CHECK (budget_limit >= 0)`
   - `created_at timestamptz NOT NULL DEFAULT now()`
   - `updated_at timestamptz NOT NULL DEFAULT now()`
   - `UNIQUE (user_id, year, month)`
   - Enable RLS with 4 policies (select/insert/update/delete) using `auth.uid() = user_id`

2. `income_sources` table:
   - `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id uuid NOT NULL`
   - `name text NOT NULL`
   - `amount numeric(14,2) NOT NULL CHECK (amount >= 0)`
   - `currency text NOT NULL DEFAULT 'USD' CHECK (currency IN ('USD', 'PHP'))`
   - `conversion_method text NOT NULL DEFAULT 'none' CHECK (conversion_method IN ('wise', 'paypal', 'none'))`
   - `payday_day int CHECK (payday_day BETWEEN 1 AND 31)`
   - `active boolean NOT NULL DEFAULT true`
   - `created_at timestamptz NOT NULL DEFAULT now()`
   - `updated_at timestamptz NOT NULL DEFAULT now()`
   - Enable RLS with 4 policies using `auth.uid() = user_id`

3. `monthly_income` table:
   - `id uuid PRIMARY KEY DEFAULT gen_random_uuid()`
   - `user_id uuid NOT NULL`
   - `month_id uuid NOT NULL REFERENCES months(id) ON DELETE CASCADE`
   - `income_source_id uuid NOT NULL REFERENCES income_sources(id) ON DELETE CASCADE`
   - `expected_amount numeric(14,2) NOT NULL`
   - `currency text NOT NULL DEFAULT 'USD'`
   - `conversion_method text NOT NULL DEFAULT 'none'`
   - `exchange_rate numeric(10,4)` (nullable — set when received)
   - `fee_amount numeric(14,2)` (nullable — set when received)
   - `net_php numeric(14,2)` (nullable — set when received)
   - `received boolean NOT NULL DEFAULT false`
   - `received_at timestamptz` (nullable)
   - `created_at timestamptz NOT NULL DEFAULT now()`
   - `updated_at timestamptz NOT NULL DEFAULT now()`
   - `UNIQUE (month_id, income_source_id)`
   - Enable RLS with 4 policies using `auth.uid() = user_id`

Add indexes:

- `income_sources_user_active_idx ON income_sources(user_id, active)`
- `monthly_income_month_id_idx ON monthly_income(month_id)`

**009_finance_rpcs.sql** — Create three RPC functions:

1. `ensure_month_exists(p_user_id uuid, p_year int, p_month int) RETURNS months` — Check if month row exists; if not, create it with `starting_balance` and `current_balance` copied from previous month's `current_balance` (or 0 if no previous month). Use `ON CONFLICT DO NOTHING` + re-select for race condition safety. `SECURITY INVOKER`, `SET search_path = public`.

2. `apply_income_received(p_monthly_income_id uuid, p_received boolean, p_net_php numeric DEFAULT NULL, p_exchange_rate numeric DEFAULT NULL, p_fee_amount numeric DEFAULT NULL) RETURNS monthly_income` — When `p_received=true`: set received=true, received_at=now(), store exchange_rate/fee_amount/net_php, add net_php to months.current_balance. When `p_received=false` (undo): subtract stored net_php from balance, clear received fields. `SECURITY INVOKER`, `SET search_path = public`.

3. `populate_monthly_income(p_user_id uuid, p_month_id uuid) RETURNS int` — Insert monthly_income rows from all active income_sources for the given month. `ON CONFLICT DO NOTHING`. Return count inserted. `SECURITY INVOKER`, `SET search_path = public`.

Use the exact SQL from the 03-RESEARCH.md code examples — it has been validated against the source app patterns.
