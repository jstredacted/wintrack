# Phase 3 — UI Design Contract

> Visual and interaction contract for the Finance Core module. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | radix-nova (neutral base) |
| Component library | Radix UI primitives |
| Icon library | Lucide React (^0.577.0) |
| Font | Geist Mono Variable (monospace for both --font-sans and --font-mono) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within income card rows |
| sm | 8px | Compact element spacing, gap between MonthStrip cells |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section gaps (balance hero to gauge, gauge to income cards) |
| xl | 32px | Page-level horizontal padding |
| 2xl | 48px | Top margin below MonthStrip before content |
| 3xl | 64px | Not used in this phase |

Exceptions: MonthStrip cells use `min-w-[5rem]` (80px) to accommodate "MMM YYYY" labels, matching DayStrip's `min-w-[4.5rem]` pattern.

---

## Typography

All sizes are in rem relative to the 18px base (`html { font-size: 18px }`). Pixel equivalents shown for reference.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 1rem (18px) | 400 (regular) | 1.5 | Income card details, budget label text, settings form labels |
| Label | 0.778rem (14px) | 400 (regular) | 1.4 | "Current Balance" label, "Monthly Budget" label, MonthStrip day abbreviations, muted secondary text |
| Heading | 1.333rem (24px) | 300 (light) | 1.2 | Section headings ("Income", budget amount inside gauge), page-level labels |
| Display | 2.667rem (48px) | 300 (light) | 1.0 | Balance hero number (the largest text on the finance page) |

Font classes: `font-mono tabular-nums` on all numeric values (balance, currency amounts, percentages). `font-light` on Display and Heading sizes. `tracking-widest uppercase` on Label text that serves as a category label (e.g., "CURRENT BALANCE").

---

## Color

This phase is fully monochrome, matching the Nothing Phone aesthetic. All colors use the existing oklch CSS custom properties.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, scroll areas |
| Secondary (30%) | `var(--card)` / `var(--muted)` | Income source cards, budget gauge background ring, MonthStrip cell backgrounds on hover |
| Accent (10%) | `var(--foreground)` at varying opacity | Selected MonthStrip border-b-2, balance hero number, budget gauge progress ring, active toggle state |
| Destructive | `var(--destructive)` | Not used in Phase 3 (no delete actions on finance page; income source deletion is in Settings with existing patterns) |

Accent reserved for:
- MonthStrip selected month `border-b-2 border-foreground`
- Balance hero number `text-foreground`
- Budget gauge progress ring `stroke: var(--foreground)` at 0.3 / 0.55 / 1.0 opacity based on threshold
- Income card "Mark Received" toggle active state `bg-foreground text-background`
- Inline balance edit cursor/underline

Budget gauge monochrome thresholds (opacity of `var(--foreground)` on the progress ring):
- Below 50% spent: opacity 0.3 (subtle)
- 50%-80% spent: opacity 0.55 (medium)
- Above 80% spent: opacity 1.0 (full)

## Component Inventory

### MonthStrip

| Property | Value |
|----------|-------|
| Pattern source | DayStrip.tsx (exact structural copy) |
| Cell size | `min-w-[5rem]` (80px), `py-4 px-3` |
| Cell content | Month abbreviation (3-letter, uppercase, `text-xs tracking-widest`), year below (`text-3xl tabular-nums font-light`) |
| Selection indicator | `border-b-2 border-foreground` on selected cell |
| Unselected state | `text-muted-foreground/40`, `border-b-2 border-transparent` |
| Scroll | `overflow-x-auto snap-x snap-mandatory scroll-smooth`, hidden scrollbar |
| Edge fades | Linear gradient from `var(--background)` to transparent, 80px wide |
| Chevron buttons | ChevronLeft / ChevronRight from Lucide, `size={16}`, appear when scrollable |
| Tap feedback | `active:scale-[0.94]` transition |

### BalanceHero

| Property | Value |
|----------|-------|
| Layout | Centered column, `text-center` |
| Label | "CURRENT BALANCE" in Label size (14px), `text-muted-foreground`, `tracking-widest uppercase` |
| Amount | Display size (48px), `font-mono tabular-nums font-light`, `text-foreground` |
| Format | Philippine Peso sign prefix, comma thousands, 2 decimal places (e.g., "PHP 45,000.00" via `Intl.NumberFormat`) |
| Edit trigger | Click/tap on the amount number itself |
| Edit mode | Amount text replaced by an `<input>` element, same font styling (`text-center font-mono tabular-nums text-[2.667rem] font-light`), `border-b-2 border-foreground`, no other borders |
| Edit confirm | Enter key or blur saves; Escape cancels and restores previous value |
| Edit cancel | Value reverts to pre-edit state, no DB write |
| Transition | No animation between view/edit modes; instant swap |

### BudgetGauge

| Property | Value |
|----------|-------|
| SVG dimensions | 160x160px |
| Stroke width | 12px |
| Background ring | `stroke: var(--border)`, full circle |
| Progress ring | `stroke: var(--foreground)`, `stroke-dasharray` / `stroke-dashoffset` technique, `stroke-linecap: round` |
| Rotation | `transform: rotate(-90deg)` so progress starts from 12 o'clock |
| Center text | Spent amount in Heading size (24px), `font-mono tabular-nums`, below it "of {limit}" in Label size (14px), `text-muted-foreground` |
| Opacity thresholds | <50%: 0.3, 50-80%: 0.55, >80%: 1.0 (on the progress ring) |
| Empty state (no budget set) | Gauge shows full background ring only, center text reads "No budget set", Label size, `text-muted-foreground` |
| Budget edit | Tap center text to set/edit budget limit (same inline edit pattern as BalanceHero but with Heading size) |

### IncomeCard (unreceived state)

| Property | Value |
|----------|-------|
| Container | `bg-card border border-border rounded-[var(--radius)]`, `p-4` (16px padding) |
| Layout | Flex column with 8px gap |
| Row 1 | Source name (`text-foreground`, Body size 18px, `font-mono`) + payday badge right-aligned (`text-muted-foreground`, Label size, "Day {N}") |
| Row 2 | Expected PHP amount (`text-foreground`, Heading size 24px, `font-mono tabular-nums font-light`) |
| Row 3 (USD sources only) | Collapsed fee summary: "{USD amount} USD at {rate}" in Label size, `text-muted-foreground`. Tap to expand. |
| Row 3 expanded | 4 rows: USD amount, exchange rate, fee deducted, net PHP. Each row is Label size with `text-muted-foreground` label and `text-foreground tabular-nums` value, right-aligned. Separated by `border-t border-border` from the collapsed summary. |
| Toggle button | Full-width button at bottom: "Mark Received" in Body size, `border border-border rounded-[var(--radius)] py-3`, `text-foreground`, `hover:bg-foreground/5`, `active:scale-[0.98]` |

### IncomeCard (received state — collapsed)

| Property | Value |
|----------|-------|
| Container | Same card container but `opacity-60` |
| Layout | Single row, `flex items-center justify-between`, `p-4` |
| Content | "{Source name}: {net PHP amount} received" in Body size, `text-muted-foreground`, `font-mono` |
| Undo button | "Undo" text button right-aligned, Label size, `text-foreground`, `underline`, `hover:no-underline` |
| Transition | Card height animates from full to collapsed using CSS `transition-all duration-200` |

### Income Sources Settings Section

| Property | Value |
|----------|-------|
| Location | SettingsPage.tsx, new section below existing settings |
| Section heading | "Income Sources" in Heading size (24px) |
| Card list | One card per source: name, amount, currency, conversion method, payday, active toggle |
| Add button | "+ Add Income Source" text button, Label size, `text-muted-foreground`, `hover:text-foreground` |
| Edit form | Inline expansion within the card (not a modal). Fields: name (text input), amount (number input), currency (select: USD/PHP), conversion method (select: Wise/PayPal/None, only shown when currency=USD), payday (number input 1-31) |
| Delete | "Remove" text button in destructive color, with confirmation: "Remove {source name}? This will not affect past months." |

## Interaction States

### MonthStrip Month Selection

| State | Visual |
|-------|--------|
| Default (unselected) | `text-muted-foreground/40`, transparent bottom border |
| Hover | `text-muted-foreground/70` |
| Selected | `text-foreground`, `border-b-2 border-foreground` |
| Active (press) | `scale-[0.94]` transform |

### Income Card Toggle

| State | Visual |
|-------|--------|
| Unreceived | Full card layout, "Mark Received" button visible |
| Loading (fetching rate) | Button text changes to "Fetching rate...", button disabled, `opacity-50` |
| Rate error | Button text changes to "Rate unavailable — retry", `text-destructive` |
| Received | Card collapses to single-line summary with "Undo" button |
| Past month (read-only) | Toggle button hidden, received state shown as static text |

### Balance Inline Edit

| State | Visual |
|-------|--------|
| Display | Static text, cursor: pointer (shows editability) |
| Editing | Input field with `border-b-2 border-foreground`, auto-focused, value selected |
| Saving | Brief `opacity-50` on the input while persisting |
| Error | Input border changes to `border-destructive`, error text below in Label size |

## Layout Contract

### Finance Page Structure

```
[MonthStrip — full width, bleeds to edges]
[Past month indicator — only shown for past months]
[Content area — max-w-[1100px] mx-auto, px-8]
  [BalanceHero — centered]
  [BudgetGauge — centered, 24px below balance]
  [Income section — 24px below gauge]
    [Section label: "INCOME"]
    [IncomeCard stack — 8px gap between cards]
    [Add income source link — only shown if zero sources configured]
```

### Page Scroll Behavior

- MonthStrip is sticky at the top of the finance page content area (not the AppShell — it scrolls with the page when there are many income cards but stays visible at the top of the finance route)
- Content below MonthStrip scrolls naturally
- No infinite scroll; all income cards render at once (maximum ~5-10 sources expected)

### Responsive Notes

- Phase 5 handles mobile responsiveness formally; Phase 3 targets desktop (1100px max-width) only
- BudgetGauge and BalanceHero center naturally at any width
- Income cards are full-width within the content area

## Animation Contract

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Income card collapse (received) | Height transition from full to single-line | 200ms | `ease-in-out` |
| Income card expand (undo) | Height transition from single-line to full | 200ms | `ease-in-out` |
| Fee detail expand/collapse | Height transition | 150ms | `ease-in-out` |
| Budget gauge progress | `stroke-dashoffset` transition on mount | 600ms | `cubic-bezier(0.16, 1, 0.3, 1)` (spring-out) |
| MonthStrip scroll | Native `scroll-smooth` | Browser default | Browser default |

No motion library animations in Phase 3 — all transitions use CSS `transition` property to avoid the tw-animate-css / motion v12 conflict documented in CLAUDE.md.
