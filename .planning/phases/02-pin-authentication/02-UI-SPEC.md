## Color

This phase uses the existing shadcn neutral palette. The PIN screen is a full-screen overlay that matches the app background -- no cards, no secondary surfaces. The aesthetic is Nothing Phone: monochrome, high contrast, minimal.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` — oklch(0.145 0 0) dark / oklch(1 0 0) light | Full-screen PIN background |
| Secondary (30%) | `var(--muted)` — oklch(0.269 0 0) dark / oklch(0.97 0 0) light | Unfilled dot circles (border only) |
| Accent (10%) | `var(--foreground)` — oklch(0.985 0 0) dark / oklch(0.145 0 0) light | Filled dots, heading text |
| Destructive | `var(--destructive)` — oklch(0.704 0.191 22.216) dark | Shake state dot color (wrong PIN) |

Accent reserved for: filled PIN dots, heading text on PIN screen, active digit brief-reveal text. Nothing else.

Blur overlay: `var(--background)` at 80% opacity + `backdrop-blur-xl` (Page Visibility blur).

Source: `src/index.css` dark mode token values. CONTEXT.md specifies "minimal, black/white, monospaced type."

## Component Inventory

### PinDots (reusable)

4 circular indicators showing PIN entry progress.

| State | Visual |
|-------|--------|
| Empty | 16px circle, 2px border `var(--muted-foreground)`, transparent fill |
| Brief reveal | 16px circle, filled `var(--foreground)`, shows digit in center for 150ms |
| Filled (masked) | 16px circle, filled `var(--foreground)`, no digit visible |
| Error (shake) | All dots turn `var(--destructive)`, entire row shakes horizontally, then clears to empty |

Layout: 4 dots in a horizontal row, 16px gap between each. Cluster is horizontally centered on screen.

### PinScreen (lock screen)

Full-screen PIN entry for returning users.

| Element | Spec |
|---------|------|
| Background | `bg-background` full viewport, no dot-grid texture |
| Title | "Enter your PIN" — heading size (24px), weight 600, centered |
| Dots | PinDots component, vertically centered in upper 40% of screen |
| Input | Hidden — no visible keypad. Listens to `keydown` events for digits 0-9, Backspace to delete |
| Error feedback | Dots shake (0.4s), turn destructive color, then reset to empty after animation ends |
| Spacing | 48px above title from top, 32px between title and dots |

### PinSetup (first-time flow)

Two-step PIN creation flow.

| Step | Title | Subtitle |
|------|-------|----------|
| 1. Enter | "Set your PIN" | "Choose a 4-digit PIN" |
| 2. Confirm | "Confirm your PIN" | "Enter it again to confirm" |
| Mismatch | "PINs didn't match" | "Let's try again" — resets to step 1 |

Layout identical to PinScreen. Title and subtitle change between steps. Subtitle uses label size (14px), muted-foreground color.

### PinGate (layout route wrapper)

Not a visible component. Renders one of: blank loading screen, PinSetup, PinScreen, or `<Outlet />`. No transitions between states — instant switch per CONTEXT.md.

### Blur Overlay (Page Visibility)

| Property | Value |
|----------|-------|
| Position | `fixed inset-0 z-50` |
| Background | `bg-background/80` |
| Blur | `backdrop-blur-xl` |
| Transition | None — instant appear/disappear (no animation) |

## Animations

All animations use plain CSS `@keyframes` in `src/index.css`. No motion library.

### pin-shake

```css
@keyframes pin-shake {
  0%, 100% { transform: translateX(0); }
  20%, 60% { transform: translateX(-8px); }
  40%, 80% { transform: translateX(8px); }
}
.pin-shake {
  animation: pin-shake 0.4s ease-in-out;
}
```

Applied to dot row on wrong PIN. On `animationend`: reset dots to empty, clear destructive color.

### pin-dot-fill

```css
@keyframes pin-dot-fill {
  0% { transform: scale(0.8); opacity: 0.5; }
  100% { transform: scale(1); opacity: 1; }
}
.pin-dot-enter {
  animation: pin-dot-fill 0.15s ease-out both;
}
```

Applied to each dot as user types a digit. Subtle scale-up on fill.

Source: CONTEXT.md "Shake animation should be subtle but clear" + RESEARCH.md Pattern 5.

## Interaction Contract

### Keyboard Input (no visible keypad)

| Key | Action |
|-----|--------|
| 0-9 | Append digit to PIN buffer (max 4) |
| Backspace | Remove last digit from buffer |
| Enter | No action (auto-submit on 4th digit) |
| All other keys | Ignored |

Auto-verify: When buffer reaches 4 digits, immediately hash and verify. No submit button.

### Digit Brief Reveal

When a digit is entered, show the actual digit character in the dot circle for 150ms, then mask to a filled circle. Only the most recently entered digit shows briefly — previous dots remain masked.

### Wrong PIN Flow

1. All 4 dots turn `var(--destructive)` color
2. Dot row plays `pin-shake` animation (0.4s)
3. On `animationend`: reset all dots to empty, clear buffer
4. Focus remains on hidden input — user can immediately retry
5. No attempt counter, no cooldown, no lockout

### PIN Setup Flow

1. Screen shows "Set your PIN" + empty dots
2. User types 4 digits (auto-advance to step 2)
3. Screen shows "Confirm your PIN" + empty dots
4. User types 4 digits again
5. If match: hash and store in Supabase, mark session unlocked, render app
6. If mismatch: show "PINs didn't match" for 1.5s, reset to step 1

### Session Lock/Unlock

| Event | Action |
|-------|--------|
| App load, no session flag | Show PinScreen (if PIN exists) or PinSetup (if no PIN) |
| Correct PIN entered | Set sessionStorage flag, render `<Outlet />` |
| 15 min idle | Clear sessionStorage flag, render PinScreen instantly |
| Tab hidden | Show blur overlay (do NOT lock) |
| Tab visible | Remove blur overlay |
| Tab closed | sessionStorage auto-clears (locked on next load) |

## Accessibility

| Concern | Solution |
|---------|----------|
| Screen reader | `aria-label="PIN entry"` on the hidden input, `role="status"` on dot indicators |
| Focus management | Auto-focus hidden input on mount; maintain focus after shake reset |
| Digit announcement | `aria-live="polite"` region announces "Digit N entered" (without revealing the digit) |
| Color contrast | Foreground on background meets WCAG AA (oklch grayscale palette, >7:1 ratio) |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` — disable shake and dot-fill animations, use instant state changes |
