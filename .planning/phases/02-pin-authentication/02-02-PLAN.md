    **PinDots.tsx** — Reusable 4-dot indicator component:

    Props: `{ digits: string; error: boolean; onShakeEnd: () => void }`
    - `digits` is a string of 0-4 characters representing entered digits
    - `error` triggers destructive color + shake animation
    - `onShakeEnd` fires on `animationend` of the shake — parent uses this to reset

    Render: A `div` with `role="status"` containing 4 dot circles in a horizontal row.
    - Container: `flex items-center justify-center gap-4` (gap-4 = 16px)
    - When `error` is true: container gets `style={{ animation: 'pin-shake 0.4s ease-in-out' }}` and `onAnimationEnd={onShakeEnd}`
    - Each dot: 16px x 16px rounded-full
    - Empty dot (index >= digits.length): `border-2 border-muted-foreground bg-transparent`
    - Filled dot (index < digits.length): `bg-foreground` with `style={{ animation: 'pin-dot-fill 0.15s ease-out both' }}`
    - Error dot: `bg-destructive` instead of `bg-foreground`
    - Brief digit reveal: Each dot has a brief-reveal mechanism. When a new digit is added (digits.length increases for that position), show the digit character in the center of the dot for 150ms using a `setTimeout`, then hide it. Use a ref to track which position was last revealed. The digit text is `text-[10px] font-mono text-background` (dark digit on light dot, inverted). After 150ms, the digit text disappears and the dot stays filled.
    - `@media (prefers-reduced-motion: reduce)`: remove all animation styles (no shake, no pin-dot-fill). Use a `useMediaQuery` check or simply apply animations only when `!window.matchMedia('(prefers-reduced-motion: reduce)').matches`.
    - Accessibility: `aria-live="polite"` on a visually-hidden span that announces "Digit {N} entered" when digits.length increases (without revealing the actual digit).

    **PinScreen.tsx** — Full-screen lock screen for returning users:

    Props: `{ onUnlock: (pin: string) => Promise<boolean> }`
    - `onUnlock` is called with the 4-digit PIN string; returns true if correct, false if wrong

    State: `digits` (string, max 4 chars), `error` (boolean)

    Layout: Full viewport `fixed inset-0 z-40 bg-background flex flex-col items-center`
    - 48px top padding (`pt-12`)
    - Title: "Enter your PIN" — `text-[1.333rem] font-semibold font-mono text-foreground text-center` (24px per UI-SPEC heading size)
    - 32px gap between title and dots (`mt-8`)
    - PinDots centered below title

    Keyboard handling: Add a `useEffect` with `keydown` listener on `document`:
    - Digits 0-9: if `digits.length < 4 && !error`, append digit
    - Backspace: if `digits.length > 0 && !error`, remove last digit
    - All other keys: ignored
    - When digits reaches length 4: immediately call `onUnlock(digits)`. If returns false, set `error = true`. On `onShakeEnd` callback from PinDots: set `error = false`, clear `digits` to empty string.
    - Auto-focus: No visible input element. The keydown listener is on document, so it captures all keyboard input regardless of focus.

    **PinSetup.tsx** — Two-step PIN creation flow:

    Props: `{ onComplete: (pin: string) => Promise<void> }`

    State: `step` ('enter' | 'confirm' | 'mismatch'), `firstPin` (string), `digits` (string), `error` (boolean)

    Layout: Same as PinScreen layout (full viewport, centered).
    - Step 'enter': Title "Set your PIN", subtitle "Choose a 4-digit PIN"
    - Step 'confirm': Title "Confirm your PIN", subtitle "Enter it again to confirm"
    - Step 'mismatch': Title "PINs didn't match", subtitle "Let's try again" — shown for 1.5 seconds, then auto-reset to 'enter' step
    - Subtitle: `text-[14px] font-mono text-muted-foreground mt-2`

    Flow:
    1. Step 'enter': User types 4 digits. Store as `firstPin`, clear `digits`, advance to step 'confirm'.
    2. Step 'confirm': User types 4 digits. Compare to `firstPin`.
       - Match: call `onComplete(firstPin)`
       - Mismatch: set step to 'mismatch', set error=true on PinDots. After shake animation ends (onShakeEnd), wait 1.5s showing the mismatch message, then reset to step 'enter' with all state cleared.

    Keyboard handling: Same pattern as PinScreen (document keydown listener).

    **PinGate.tsx** — Layout route that gates all app content:

    ```typescript
    import { useEffect } from 'react';
    import { Outlet } from 'react-router';
    import { usePinStore } from '@/stores/pinStore';
    import { usePinAuth } from '@/hooks/usePinAuth';
    import { useIdleTimer } from '@/hooks/useIdleTimer';
    import PinScreen from './PinScreen';
    import PinSetup from './PinSetup';
    ```

    On mount (useEffect, runs once):
    1. Call `usePinAuth().initializeGate()` — this determines if we show setup, lock, or unlock
    2. Set up Page Visibility API listener: `document.addEventListener('visibilitychange', handler)`
       - When `document.hidden`: `usePinStore.getState().setBlurred(true)`
       - When visible: `usePinStore.getState().setBlurred(false)`
       - Cleanup on unmount

    Use `useIdleTimer(() => usePinStore.getState().lock(), gateState === 'unlocked')` — only active when unlocked.

    Render based on `gateState` from `usePinStore`:
    - `'loading'`: `<div className="fixed inset-0 bg-background" />` — blank screen, prevents content flash
    - `'setup'`: `<PinSetup onComplete={async (pin) => { await pinAuth.setup(pin); }} />`
    - `'locked'`: `<PinScreen onUnlock={async (pin) => { const ok = await pinAuth.verify(pin); if (ok) usePinStore.getState().markUnlocked(); return ok; }} />`
    - `'unlocked'`: Render `<Outlet />` plus the blur overlay

    Blur overlay (only when `gateState === 'unlocked'`):
    ```tsx
    {blurred && (
      <div className="fixed inset-0 z-50 bg-background/80 backdrop-blur-xl" />
    )}
    ```
    No transition, no animation — instant appear/disappear per UI-SPEC.

    CRITICAL: Do NOT render `<Outlet />` in any state other than 'unlocked'. This prevents child components from mounting and fetching data while locked.
