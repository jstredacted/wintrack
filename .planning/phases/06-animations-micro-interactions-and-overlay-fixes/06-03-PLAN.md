Key: The outer portal div already uses CSS `journal-overlay-enter`/`journal-overlay-exit` on `transform`. Inner `motion.div` uses CSS `translate` property. These are DIFFERENT DOM elements — safe to combine per research.

Critical constraint from RESEARCH.md pitfall 1:
The outer portal div's `onAnimationEnd` fires for ALL animationend events that bubble from children.
The existing code does NOT have an `e.target === e.currentTarget` guard.
After adding motion.div children, the inner motion animations will bubble and fire the onAnimationEnd, which could prematurely call `setVisible(false)`.
YOU MUST add the bubble guard:
```jsx
onAnimationEnd={(e) => {
  if (e.target !== e.currentTarget) return; // guard against bubbling from child motion elements
  if (exiting) { setVisible(false); setScreen('editing'); }
}}
```
